1. 규칙 표현 방식으로 어떤 옵션을 선택했는가? 그 이유는?
-> Java의 Predicate<Message>를 사용함. 미리 컴파일된 Java 바이트 코드로 실행되어 스크립트 파싱이나 리플렉션 없이
실행 속도가 가장 빠름.

2. 규칙이 복잡해지면 노드 하나에 넣을 것인가, 여러 노드를 조합할 것인가?
-> 여러 노드의 조합으로 할 것임. 단일 책임 원칙에 위배될 수 있기 때문임. 여러 노드로 조합할 경우 유지 보수가 유리해짐

3. 시간 기반 규칙에서 메시지 누락(네트워크 지연 등)은 어떻게 고려하는가?
-> 메시지를 생성할 때 페이로드에 timestamp를 기록해서 전송함. 엔진은 시스템 시간이 아닌 큐에 담긴 메시지의 timestamp을 읽어야함.
-> 지연 도착 데이터를 위해 윈도우를 즉시 닫지 않고 지연 데이터 도착 시간을 줌.

4. 규칙을 외부 파일(JSON/YAML)로 정의하고 런타임에 로드하는 방법은?
-> JSON/YAML의 포멧을 만든 후 파서를 구현하여 가져옴.
